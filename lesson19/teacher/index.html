0〜25分
💬
講師がこう話す「第10回でDOMの要素追加を学びましたが、今回はより高度なテクニックを扱います。大量の要素を効率よく生成する方法・テンプレート要素を使う方法・DocumentFragmentでパフォーマンスを上げる方法を学びます。」
script.js（DocumentFragmentで高速挿入）
// 普通の方法：1つずつappendChildすると遅い const ul = document.querySelector("#list"); for (let i = 0; i < 100; i++) { const li = document.createElement("li"); li.textContent = `アイテム ${{i + 1}}`; ul.appendChild(li); // 100回DOMを更新 → 遅い！ } // DocumentFragmentを使う方法：1回でDOMに挿入 const fragment = document.createDocumentFragment(); for (let i = 0; i < 100; i++) { const li = document.createElement("li"); li.textContent = `アイテム ${{i + 1}}`; fragment.appendChild(li); // fragmentに追加（DOMは未更新） } ul.appendChild(fragment); // 最後に1回だけDOMを更新 → 速い！
script.js（template要素で雛形を使いまわす）
<!-- HTML側にtemplate要素を定義 --> <template id="card-template"> <div class="card"> <h2 class="card-title"></h2> <p class="card-body"></p> <button class="card-btn">詳細を見る</button> </div> </template> // template要素をJSでクローンして使う const tmpl = document.querySelector("#card-template"); const container = document.querySelector("#cards"); const items = [ { title: "カード1", body: "説明文1" }, { title: "カード2", body: "説明文2" }, ]; items.forEach(item => { const clone = tmpl.content.cloneNode(true); // 深いコピー clone.querySelector(".card-title").textContent = item.title; clone.querySelector(".card-body").textContent = item.body; container.appendChild(clone); });
insertAdjacentHTML vs createElement
使い分けの基準
insertAdjacentHTML：HTMLを文字列でまとめて挿入したいとき。テンプレートリテラルと相性が良い。XSSに注意（ユーザー入力を入れない）。

createElement：要素1つ1つをJSで組み立てるとき。XSSリスクがない。イベントリスナーをすぐ付けたいとき。

template要素：同じHTML構造を何度も使いまわすとき。デザイナー・マークアップエンジニアとの分業に向く。
👆
講師メモ大量の要素を追加するとき、ループ内でappendChildを繰り返すとリフローが毎回発生して遅い。DocumentFragmentかinnerHTMLの一括代入が高速。パフォーマンスの話は興味を持つ生徒には深掘りして良い。

data属性・座標取得・MutationObserver
25〜45分
💬
講師がこう話す「data属性はHTML要素に独自のデータを持たせる方法です。イベントデリゲーションと組み合わせると、どの要素が操作されたかを簡単に判別できます。getBoundingClientRectは要素の位置・サイズを取得でき、ツールチップやドロップダウンの実装に使います。」
script.js（data属性）
<!-- HTML側でdata-*属性を定義 --> <button class="product-btn" data-id="42" data-name="ノートPC" data-price="98000"> カートに追加 </button> // JSでdata属性を取得・設定する const btn = document.querySelector(".product-btn"); // datasetプロパティ経由でアクセス（data-を除いたキャメルケース） console.log(btn.dataset.id); // "42"（文字列） console.log(btn.dataset.name); // "ノートPC" console.log(btn.dataset.price); // "98000" // 数値が必要なときは変換する const productId = Number(btn.dataset.id); // 42 // data属性を設定・削除 btn.dataset.quantity = "1"; // 設定 delete btn.dataset.quantity; // 削除
script.js（getBoundingClientRect・ポジション計算）
// getBoundingClientRect：要素のサイズと位置を取得 const el = document.querySelector(".target"); const rect = el.getBoundingClientRect(); console.log(rect.top); // ビューポート上端からの距離 console.log(rect.left); // ビューポート左端からの距離 console.log(rect.width); // 要素の幅 console.log(rect.height); // 要素の高さ console.log(rect.bottom); // ビューポート上端からの距離 + 高さ // ビューポート内に表示されているか判定 const isVisible = (element) => { const r = element.getBoundingClientRect(); return r.top >= 0 && r.bottom <= (window.innerHeight || document.documentElement.clientHeight); }; // ツールチップの位置をボタンの位置に合わせる例 btn.addEventListener("mouseenter", () => { const r = btn.getBoundingClientRect(); tooltip.style.top = `${{r.bottom + window.scrollY + 8}}px`; tooltip.style.left = `${{r.left + window.scrollX}}px`; tooltip.classList.add("show"); });
💡
getBoundingClientRectの座標はビューポート基準です。ページ全体の絶対位置を得るには rect.top + window.scrollY を使います。スクロール量を足すのを忘れると、スクロール後にズレます。
