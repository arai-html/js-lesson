配列を作って操作しよう
0〜25分
📖
配列は複数の値をまとめて管理するリストです。インデックス（番号）で各要素にアクセスします。
script.js（配列の基本）
// 配列の作成（[]で囲む） const fruits = ["りんご", "バナナ", "みかん"]; const scores = [85, 72, 90, 68]; const mixed = ["太郎", 25, true]; // 違う型も混在できる // 要素へのアクセス（インデックスは0始まり） console.log(fruits[0]); // "りんご" console.log(fruits[2]); // "みかん" console.log(fruits.length); // 3（要素数） // 最後の要素を取り出す console.log(fruits[fruits.length - 1]); // "みかん" console.log(fruits.at(-1)); // "みかん"（新しい書き方） // 要素の追加・削除 fruits.push("ぶどう"); // 末尾に追加 → ["りんご","バナナ","みかん","ぶどう"] fruits.pop(); // 末尾を削除 → ["りんご","バナナ","みかん"] fruits.unshift("いちご"); // 先頭に追加 fruits.shift(); // 先頭を削除
pushとpopをセットで覚える
スタック（LIFO）の操作
push()：末尾に追加 / pop()：末尾から取り出し
unshift()：先頭に追加 / shift()：先頭から取り出し

実務ではpush()が最もよく使います。動的なリスト生成（ToDoアプリなど）では欠かせません。
2
便利な配列メソッドをマスターしよう
25〜40分
📖
map・filter・findは配列操作の三種の神器です。for文より短くスッキリ書けます。
script.js（map・filter・find）
const scores = [85, 72, 90, 68, 95]; // map()：全要素を変換した新しい配列を返す const doubled = scores.map(score => score * 2); console.log(doubled); // [170, 144, 180, 136, 190] const grades = scores.map(score => score >= 80 ? "合格" : "不合格"); console.log(grades); // ["合格","不合格","合格","不合格","合格"] // filter()：条件に合う要素だけを取り出した新しい配列を返す const passed = scores.filter(score => score >= 80); console.log(passed); // [85, 90, 95] // find()：条件に合う最初の1要素を返す（なければundefined） const first90 = scores.find(score => score >= 90); console.log(first90); // 90 // includes()：指定した値が含まれるか（true/false） console.log(scores.includes(72)); // true console.log(scores.includes(100)); // false // reduce()：配列を1つの値に集約する const total = scores.reduce((sum, score) => sum + score, 0); console.log(`合計：${{total}}、平均：${{total / scores.length}}`);
💡
map・filter・findはオリジナルの配列を変更しません。新しい配列（またはひとつの値）を返します。これを「非破壊的メソッド」と言います。一方、push・pop・sortは元の配列を変更する「破壊的メソッド」です。
