同期処理と非同期処理の違い
0〜15分
💬
講師がこう話す「JavaScriptは基本的に1行ずつ順番に実行されます（同期処理）。でもAPIからデータを取得するとき、返ってくるまでずっと待ってたら画面が固まってしまいます。そこで非同期処理の出番です。処理を別に任せて、完了したら結果を受け取る仕組みです。」
⏱
同期と非同期のイメージ
同期処理（順番待ち）：ラーメン屋でカウンター注文。作り終わるまで次の人は注文できない。

非同期処理（並行）：ファミレスで注文。注文したらすぐ次のお客さんの注文を取り、できた料理を後から届ける。

APIのデータ取得・ファイルの読み込み・タイマー処理などは全て非同期で行います。
script.js（同期 vs 非同期）
// 同期処理：上から順番に実行される console.log("① 最初"); console.log("② 次"); console.log("③ 最後"); // → ① ② ③ の順に表示 // 非同期処理：setTimeoutは後回しにされる console.log("① 最初"); setTimeout(() => { console.log("② 1秒後"); // 1000ms後に実行 }, 1000); console.log("③ すぐ実行"); // → ① ③ ② の順に表示！（②は後回し）
👆
講師メモ実際にコンソールで実行して順番を確認させる。「え、なんで？」という驚きが非同期理解の第一歩。図を描いて説明するとより効果的。

setTimeout・setInterval
15〜30分
💬
講師がこう話す「setTimeoutは一定時間後に1回だけ処理を実行します。setIntervalは一定間隔で繰り返し実行します。どちらもIDを返すので、clearTimeout/clearIntervalでキャンセルできます。」
script.js（setTimeout・setInterval）
// setTimeout：指定時間後に1回だけ実行 const timerId = setTimeout(() => { console.log("3秒経ちました！"); }, 3000); // 3000ms = 3秒 // キャンセルしたい場合 clearTimeout(timerId); // setInterval：一定間隔で繰り返し実行 let count = 0; const intervalId = setInterval(() => { count++; console.log(`${{count}}秒経過`); if (count >= 5) { clearInterval(intervalId); // 5回で停止 console.log("カウント終了！"); } }, 1000); // 実践例：カウントダウンタイマー const display = document.querySelector("#timer"); let remaining = 10; const timer = setInterval(() => { display.textContent = remaining; remaining--; if (remaining < 0) { clearInterval(timer); display.textContent = "終了！"; } }, 1000);
setTimeoutとsetIntervalの使い分け
1回か繰り返しかで選ぶ
setTimeout：一定時間後に1回だけ実行
使い道：通知を数秒後に消す・ローディング後に画面を表示する

setInterval：一定間隔で繰り返し実行
使い道：カウントダウン・スライダーの自動再生・時計の表示

どちらもclearTimeout/clearIntervalで止めることを忘れずに！

Promiseの基本
30〜45分
💬
講師がこう話す「Promiseは非同期処理の結果（成功・失敗）を扱うオブジェクトです。コールバック地獄と呼ばれる入れ子のコードを解消するために生まれました。次回のasync/awaitを理解するための土台になります。」
script.js（Promiseの基本）
// Promiseを作る（resolve=成功・reject=失敗） const fetchData = () => { return new Promise((resolve, reject) => { setTimeout(() => { const success = true; // 成功シミュレーション if (success) { resolve({ name: "田中", age: 25 }); // 成功：データを返す } else { reject(new Error("取得失敗")); // 失敗：エラーを返す } }, 1000); }); }; // Promiseを使う（.then()で成功・.catch()で失敗を処理） fetchData() .then(data => { console.log("成功：", data); // {name:"田中", age:25} }) .catch(error => { console.log("エラー：", error.message); }) .finally(() => { console.log("成功・失敗に関わらず実行"); // ローディング解除など });
💡
Promiseには3つの状態があります。pending（処理中）→ fulfilled（成功）または rejected（失敗）。一度fulfilled/rejectedになると状態は変わりません。.finally()は成功・失敗どちらでも実行されるので、ローディング表示の解除などに使います。
