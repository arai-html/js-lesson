useEffectを深く理解しよう
0〜25分
📖
useEffectの依存配列と、タイマー・イベントを解除するクリーンアップ関数を学びます。
app.jsx（依存配列・クリーンアップ）
const { useState, useEffect } = React; const SearchResults = () => { const [query, setQuery] = useState(""); const [results, setResults] = useState([]); // queryが変わるたびに実行（依存配列に[query]） useEffect(() => { if (!query) { setResults([]); return; } // debounce代わりにタイマーを使う const timer = setTimeout(async () => { const res = await fetch( `https://jsonplaceholder.typicode.com/posts?title_like=${{query}}` ); const data = await res.json(); setResults(data.slice(0, 5)); }, 300); // クリーンアップ関数：次のeffect実行前やアンマウント時に呼ばれる return () => clearTimeout(timer); // タイマーをキャンセル }, [query]); // queryが変わるたびに実行 return ( <div> <input value={query} onChange={e => setQuery(e.target.value)} placeholder="検索..." /> <ul> {results.map(r => <li key={r.id}>{r.title}</li>)} </ul> </div> ); };
app.jsx（カスタムHook）
// カスタムHook：Hookのロジックを再利用可能な関数として切り出す // 命名規則：useName で始める const useFetch = (url) => { const [data, setData] = useState(null); const [loading, setLoading] = useState(true); const [error, setError] = useState(null); useEffect(() => { let cancelled = false; // アンマウント後の更新を防ぐ setLoading(true); fetch(url) .then(r => r.json()) .then(d => { if (!cancelled) setData(d); }) .catch(e => { if (!cancelled) setError(e.message); }) .finally(() => { if (!cancelled) setLoading(false); }); return () => { cancelled = true; }; // クリーンアップ }, [url]); return { data, loading, error }; }; // 使い方：URL を渡すだけで data/loading/error が返る const Posts = () => { const { data, loading, error } = useFetch( "https://jsonplaceholder.typicode.com/posts?_limit=5" ); if (loading) return <p>読み込み中...</p>; if (error) return <p>エラー: {error}</p>; return <ul>{data?.map(p => <li key={p.id}>{p.title}</li>)}</ul>; };
カスタムHookのメリット
ロジックの再利用と関心の分離
useFetchを一度作っておけば、どのコンポーネントでもconst {'{'} data, loading, error {'}'} = useFetch(url)の1行でAPI通信が使えます。

「データ取得のロジック」と「UIの表示」を分離できるので、コードが読みやすくテストしやすくなります。カスタムHookはuse〇〇という命名が必須です。

useContextでpropsのバケツリレーを解消
25〜45分
📖
useContextはpropsのバケツリレーを解消して、どの階層からでもデータを受け取れる仕組みです。
app.jsx（useContext）
const { useState, useContext, createContext } = React; // Contextを作る const ThemeContext = createContext("light"); // 深くネストしたコンポーネント const DeepChild = () => { const theme = useContext(ThemeContext); // propsなしで受け取れる！ return ( <div style={{{{ background: theme === "dark" ? "#1a1a1a" : "#fff", color: theme === "dark" ? "#fff" : "#000", padding: "16px" }}}}> 現在のテーマ: {theme} </div> ); }; const App = () => { const [theme, setTheme] = useState("light"); return ( // Providerでラップ → valueで値を配布 <ThemeContext.Provider value={theme}> <button onClick={() => setTheme(t => t === "light" ? "dark" : "light") }> テーマ切り替え </button> <DeepChild /> </ThemeContext.Provider> ); };
💡
useContextはglobal stateの一形態です。しかし全てのstateをContextにするのはアンチパターンです。「どこからでもアクセスしたいもの（テーマ・言語・ユーザー情報）だけContextに入れる」のが正しい使い方です。頻繁に変わるstateをContextに入れると不要な再レンダリングが起きてパフォーマンスが落ちます。
