高階関数とコールバックを理解しよう
0〜25分
📖
高階関数は関数を引数に受け取ったり返したりする関数です。map・filter・addEventListenerも高階関数です。
💡
実はもう使っていた！
arr.map(fn)・arr.filter(fn)・arr.forEach(fn)・addEventListener('click', fn)

これらは全て高階関数です。関数（fn）を引数として受け取っています。知らないうちにすでに使いこなしていました！
script.js（コールバック関数）
// コールバック関数：引数として渡される関数 const doOperation = (a, b, callback) => { const result = callback(a, b); console.log(`結果: ${{result}}`); }; doOperation(10, 5, (a, b) => a + b); // 結果: 15 doOperation(10, 5, (a, b) => a * b); // 結果: 50 doOperation(10, 5, (a, b) => a - b); // 結果: 5 // 配列のソートもコールバックで制御できる const nums = [3, 1, 4, 1, 5, 9]; const users = [ { name: "太郎", age: 25 }, { name: "花子", age: 22 }, { name: "次郎", age: 28 }, ]; nums.sort((a, b) => a - b); // 昇順 [1,1,3,4,5,9] users.sort((a, b) => a.age - b.age); // 年齢昇順 users.sort((a, b) => b.age - a.age); // 年齢降順 users.sort((a, b) => a.name.localeCompare(b.name)); // 名前順
script.js（関数を返す関数）
// 関数を返す関数（ファクトリー関数） const createMultiplier = (factor) => { return (num) => num * factor; // 関数を返す }; const double = createMultiplier(2); const triple = createMultiplier(3); console.log(double(5)); // 10 console.log(triple(5)); // 15 // 実用例：バリデーション関数を生成するファクトリー const createValidator = (minLen, maxLen) => { return (value) => value.length >= minLen && value.length <= maxLen; }; const isValidName = createValidator(2, 20); const isValidPassword = createValidator(8, 50); console.log(isValidName("太郎")); // true console.log(isValidPassword("abc")); // false（短すぎ）
コールバック地獄を避けるには
ネストを浅く保つ
コールバックが何重にも入れ子になる「コールバック地獄」は可読性を下げます。

対策①：関数を先に名前付きで定義してからコールバックに渡す
対策②：Promise・async/awaitで非同期のネストを解消する
対策③：処理を小さな関数に分割してネストを浅くする

クロージャ・debounce・memoize
25〜45分
📖
クロージャは外側のスコープの変数を記憶する関数の性質です。debounceなど実用的なパターンで活用できます。
script.js（クロージャ）
// クロージャ：内側の関数が外側の変数を「記憶」する const createCounter = () => { let count = 0; // この変数は返された関数に「記憶」される return { increment: () => ++count, decrement: () => --count, reset: () => { count = 0; }, get: () => count, }; }; const counter = createCounter(); console.log(counter.increment()); // 1 console.log(counter.increment()); // 2 console.log(counter.decrement()); // 1 console.log(counter.get()); // 1 // 別のインスタンスはそれぞれ独立した count を持つ const counter2 = createCounter(); counter2.increment(); // counter2は独立 → 1 console.log(counter.get()); // counter は影響なし → 1
script.js（debounce：実務必須テクニック）
// debounce：連続して呼ばれても最後の呼び出しから一定時間後に1回だけ実行 // 検索ボックス・リサイズイベントなどで必須 const debounce = (fn, delay) => { let timer; return (...args) => { clearTimeout(timer); timer = setTimeout(() => fn(...args), delay); }; }; // 使い方：検索ボックスへの入力（毎回APIを叩かないようにする） const searchAPI = debounce((query) => { console.log(`"${{query}}"で検索`); // ここでfetchする }, 300); // 入力が止まって300ms後に実行 document.querySelector("#search")?.addEventListener("input", (e) => { searchAPI(e.target.value); // 連打してもAPI呼び出しは300ms後に1回 });
💡
debounceとthrottleは混同されやすいです。debounceは「入力が止まってからN秒後に実行」（検索・バリデーションに使う）。throttleは「N秒間に最大1回実行」（スクロール・リサイズに使う）。どちらもパフォーマンス改善の重要テクニックです。
