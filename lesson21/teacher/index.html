スクロールイベントを使いこなそう
0〜25分
💬
講師がこう話す「スクロールイベントはページをスクロールするたびに発火します。ただしscrollイベントは非常に高頻度で発火するので、そのままAPI呼び出しやDOM操作をすると重くなります。throttle・requestAnimationFrame・IntersectionObserverを使って賢く対処しましょう。」
script.js（scrollイベントの基本）
// scrollイベント：スクロールのたびに発火 window.addEventListener("scroll", () => { const scrollY = window.scrollY; // スクロール量（px） const total = document.body.scrollHeight - window.innerHeight; const percent = (scrollY / total * 100).toFixed(1); console.log(`スクロール量: ${{scrollY}}px (${{percent}}%)`); }); // スクロールプログレスバー const bar = document.querySelector("#progress-bar"); window.addEventListener("scroll", () => { const pct = window.scrollY / (document.body.scrollHeight - window.innerHeight) * 100; bar.style.width = `${{pct}}%`; }); // スティッキーヘッダー（スクロールでclassを付け外し） const header = document.querySelector("header"); window.addEventListener("scroll", () => { header.classList.toggle("scrolled", window.scrollY > 80); });
script.js（throttleでパフォーマンス改善）
// throttle：N ms に1回だけ実行する（スクロールに最適） const throttle = (fn, limit) => { let lastTime = 0; return (...args) => { const now = Date.now(); if (now - lastTime >= limit) { lastTime = now; fn(...args); } }; }; // 100ms に1回だけ実行（scrollイベントを間引く） window.addEventListener("scroll", throttle(() => { console.log("間引いたscroll", window.scrollY); }, 100)); // requestAnimationFrame：次の描画タイミングで実行（なめらか） let ticking = false; window.addEventListener("scroll", () => { if (!ticking) { requestAnimationFrame(() => { // ここで描画処理 ticking = false; }); ticking = true; } });
debounce・throttle・rAFの使い分け
スクロール最適化の3つの武器
debounce：入力が止まってからN秒後に実行。スクロール停止後の処理に。
throttle：N msに1回だけ実行。スクロール中の継続的な処理に。
requestAnimationFrame：次のフレーム描画時に実行。アニメーションに最適（60fps）。

スクロールでDOM操作するならrAFが最もなめらかです。
👆
講師メモパフォーマンスの話は生徒によって興味の差が大きい。基本はthrottleを使えばOKと伝えて、興味ある生徒にはrAFを深掘りする。
2
IntersectionObserverでスクロールアニメーション
25〜45分
💬
講師がこう話す「IntersectionObserverは要素がビューポートに入ったかどうかを監視するAPIです。scrollイベントと違い、要素が見えたときだけ発火するので非常に効率的です。スクロールアニメーション・遅延読み込みの標準的な実装方法です。」
script.js（IntersectionObserver）
// IntersectionObserver：要素がビューポートに入ったとき発火 const observer = new IntersectionObserver((entries) => { entries.forEach(entry => { if (entry.isIntersecting) { // 要素がビューポートに入った entry.target.classList.add("visible"); observer.unobserve(entry.target); // 一度発火したら監視解除 } }); }, { threshold: 0.1, // 10%見えたら発火 rootMargin: "0px 0px -50px 0px" // 下から50px手前で発火 }); // 監視する要素を登録 document.querySelectorAll(".fade-in").forEach(el => { observer.observe(el); });
style.css（フェードインアニメーション）
/* 初期状態：透明で少し下にオフセット */ .fade-in { opacity: 0; transform: translateY(30px); transition: opacity 0.6s ease, transform 0.6s ease; } /* JSでvisibleクラスが付いたら表示 */ .fade-in.visible { opacity: 1; transform: translateY(0); } /* 遅延を使って順番にアニメーション */ .fade-in:nth-child(1) { transition-delay: 0s; } .fade-in:nth-child(2) { transition-delay: 0.1s; } .fade-in:nth-child(3) { transition-delay: 0.2s; } .fade-in:nth-child(4) { transition-delay: 0.3s; }
💡
IntersectionObserverはIE非対応ですが、現代のブラウザでは全て対応しています。unobserve()で監視を解除するのを忘れずに。解除しないと一度見えた要素を監視し続けてメモリの無駄になります。
