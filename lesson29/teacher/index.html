デバッグをマスターしよう
0〜25分
💬
講師がこう話す「プロのエンジニアも毎日バグと戦っています。大切なのはバグに出会ったとき焦らず原因を特定する手順を持つことです。デベロッパーツールのブレークポイント・console系メソッド・よくあるバグパターンを体系的に学びます。」
script.js（よくあるバグと解決策①）
// ❌ バグ①：undefined のプロパティにアクセス const user = null; console.log(user.name); // TypeError: Cannot read properties of null // ✅ 解決：オプショナルチェーンでガード console.log(user?.name); // undefined（エラーにならない） // ❌ バグ②：非同期処理の順序ミス let data; fetch("https://jsonplaceholder.typicode.com/users/1") .then(r => r.json()) .then(d => { data = d; }); console.log(data); // undefined！（fetchはまだ完了していない） // ✅ 解決：await で待つ const load = async () => { const res = await fetch("https://jsonplaceholder.typicode.com/users/1"); const data = await res.json(); console.log(data); // 正しく取得できる }; // ❌ バグ③：配列メソッドの戻り値を使い忘れ const nums = [1, 2, 3]; nums.map(n => n * 2); // 戻り値を捨てている！ console.log(nums); // [1, 2, 3]（変わっていない） // ✅ 解決：戻り値を変数に代入 const doubled = nums.map(n => n * 2); console.log(doubled); // [2, 4, 6]
script.js（よくあるバグと解決策②）
// ❌ バグ④：===ではなく==を使った比較 console.log(0 == ""); // true（型変換される！） console.log(null == undefined); // true console.log(1 == "1"); // true // ✅ 解決：常に=== を使う console.log(1 === "1"); // false（型も一致しないとfalse） // ❌ バグ⑤：sort()が数値を文字列として並べる const arr = [10, 9, 2, 100, 21]; console.log(arr.sort()); // [10, 100, 2, 21, 9]（文字列順！） console.log(arr.sort((a,b) => a - b)); // [2, 9, 10, 21, 100]（数値順） // ❌ バグ⑥：オブジェクトの参照コピー const orig = { name: "太郎", age: 25 }; const copy = orig; // 参照コピー！ copy.age = 99; console.log(orig.age); // 99（origも変わってしまった！） // ✅ 解決：スプレッド構文でシャローコピー const copy2 = { ...orig }; copy2.age = 99; console.log(orig.age); // 25（origは変わらない）
デバッグの基本手順
焦らず3ステップで
① エラーメッセージを読む：行番号・エラー種別を確認
② 原因を仮定する：「どこで何が起きているか」を考える
③ console.log・ブレークポイントで確認：仮定を検証する

デベロッパーツールのSourcesタブ→行番号クリックでブレークポイント設定→変数の中身をリアルタイムで確認できます。

パフォーマンス最適化
25〜45分
💬
講師がこう話す「パフォーマンスとは処理速度とメモリ効率のことです。学習段階では深く考えなくていいですが、基本的な考え方を知っておくとプロのコードを読んだときに意図が分かります。」
script.js（パフォーマンス改善のポイント）
// ① DOM操作は最小限に（バッチ処理） // ❌ ループ内で毎回DOM操作 const list = document.querySelector("ul"); ["A","B","C"].forEach(item => { list.innerHTML += `<li>${{item}}</li>`; // 毎回リフロー発生！ }); // ✅ 一括でinnerHTMLを更新 list.innerHTML = ["A","B","C"].map(i => `<li>${{i}}</li>`).join(""); // ② 重い処理をdebounce/throttleで間引く（第18回参照） // ③ 不要なイベントリスナーを解除する const handler = () => console.log("clicked"); document.addEventListener("click", handler); // 不要になったら解除 document.removeEventListener("click", handler); // ④ 画像の遅延読み込み（loading="lazy"） // <img src="photo.jpg" loading="lazy" alt=""> // ← ビューポートに近づいたとき初めて読み込む // ⑤ console.timeで処理時間を計測 console.time("重い処理"); const result = Array.from({ length: 100000 }, (_, i) => i * 2); console.timeEnd("重い処理"); // "重い処理: X.XXms" // ⑥ Lighthouseでページ全体のパフォーマンスを計測 // DevTools → Lighthouseタブ → Analyze page load
💡
ChromeのLighthouseツール（DevTools→Lighthouseタブ）はページのパフォーマンス・SEO・アクセシビリティ・ベストプラクティスを自動で採点してくれます。ポートフォリオサイトをLighthouseで90点以上にしておくと就職活動でのアピールポイントになります。
