useStateで状態を管理しよう
0〜25分
💬
講師がこう話す「stateはコンポーネントが持つ変化するデータです。stateが変わるとReactが自動でそのコンポーネントを再描画します。useStateはstateを作るHookです。Hookとはuse〇〇という関数コンポーネントの中だけで使える特別な関数です。」
app.jsx（useStateの基本）
import { useState } from "react"; // CDNの場合は const { useState } = React; const Counter = () => { // [現在の値, 更新する関数] = useState(初期値) const [count, setCount] = useState(0); return ( <div> <p>カウント: {count}</p> <button onClick={() => setCount(count + 1)}>+1</button> <button onClick={() => setCount(count - 1)}>-1</button> <button onClick={() => setCount(0)}>リセット</button> </div> ); };
app.jsx（フォームとstate）
const NameForm = () => { const [name, setName] = useState(""); const [greet, setGreet] = useState(""); const handleSubmit = (e) => { e.preventDefault(); setGreet(`こんにちは、${{name}}さん！`); }; return ( <div> <form onSubmit={handleSubmit}> <input value={name} onChange={(e) => setName(e.target.value)} placeholder="名前を入力" /> <button type="submit">挨拶</button> </form> {greet && <p>{greet}</p>} </div> ); };
app.jsx（配列state・ToDoリスト）
const TodoApp = () => { const [todos, setTodos] = useState([]); const [input, setInput] = useState(""); const addTodo = () => { if (!input.trim()) return; // stateの配列は必ず新しい配列に！（スプレッド構文） setTodos([...todos, { id: Date.now(), text: input, done: false }]); setInput(""); }; const toggleTodo = (id) => setTodos(todos.map(t => t.id === id ? { ...t, done: !t.done } : t)); const deleteTodo = (id) => setTodos(todos.filter(t => t.id !== id)); return ( <div> <input value={input} onChange={e => setInput(e.target.value)} /> <button onClick={addTodo}>追加</button> <ul> {todos.map(todo => ( <li key={todo.id} style={{{{ textDecoration: todo.done ? "line-through" : "none" }}}}> <span onClick={() => toggleTodo(todo.id)}>{todo.text}</span> <button onClick={() => deleteTodo(todo.id)}>削除</button> </li> ))} </ul> </div> ); };
stateの更新は直接変更NG
必ず新しい値・配列・オブジェクトを渡す
Reactのstateを直接変更してもUIは更新されません。

❌ todos.push(newItem)（直接変更）
✅ setTodos([...todos, newItem])（新配列）

❌ user.age = 26
✅ setUser({'{'} ...user, age: 26 {'}'})

「stateは常に新しい値で上書き」が鉄則です。
👆
講師メモstateを直接変更してしまうバグは初学者が最も多くハマるポイント。pushで追加したのにUIが変わらない経験を一度させて「だからsetStateに新配列を渡すんだ」と体感させると定着する。
2
useEffectでAPI通信をしよう
25〜45分
💬
講師がこう話す「useEffectはコンポーネントのマウント時・stateの変化時に副作用（API通信・タイマー・イベント登録など）を実行するHookです。第2引数の依存配列で『いつ実行するか』を制御します。」
app.jsx（useEffectの基本）
const { useState, useEffect } = React; const UserList = () => { const [users, setUsers] = useState([]); const [loading, setLoading] = useState(true); const [error, setError] = useState(null); // マウント時に1回だけ実行（依存配列が[]） useEffect(() => { const fetchUsers = async () => { try { const res = await fetch("https://jsonplaceholder.typicode.com/users"); const data = await res.json(); setUsers(data); } catch (e) { setError(e.message); } finally { setLoading(false); } }; fetchUsers(); }, []); // ← []でマウント時の1回だけ if (loading) return <p>読み込み中...</p>; if (error) return <p>エラー: {error}</p>; return ( <ul> {users.map(u => <li key={u.id}>{u.name} — {u.email}</li>)} </ul> ); };
💡
useEffectの第2引数（依存配列）の使い分け：
• []：マウント時に1回だけ
• [id]：idが変わるたびに実行
• 省略：毎回レンダリング後に実行（ほぼ使わない）

API通信は[]が定番です。
