TypeScriptの基本を学ぼう
0〜25分
💬
講師がこう話す「TypeScriptはJavaScriptに型を加えた言語で、Microsoftが開発しました。TypeScriptを書いてコンパイルするとJavaScriptが生成されます。型があることで、エラーを事前に発見できる・コードの補完が効く・チームで開発しやすいという大きなメリットがあります。現代のフロントエンド開発では標準的に使われています。」
🔷
TypeScriptのメリット
① 型安全：引数の型ミスをコンパイル時に発見できる
② IDE補完：VSCodeで強力なコード補完・ドキュメント表示
③ リファクタリング：変数名変更時に影響箇所を自動検出
④ チーム開発：型が仕様書の役割を果たす

React・Vue・Angularなど主要フレームワークは全てTypeScript対応
TypeScript（基本の型アノテーション）
// 基本の型 let name: string = "田中"; let age: number = 25; let isAdmin: boolean = false; let tags: string[] = ["JS", "TS"]; // 文字列の配列 // 型が違うと代入でエラー name = 42; // ❌ Type 'number' is not assignable to type 'string' // 関数の型アノテーション const add = (a: number, b: number): number => { return a + b; }; add(1, "2"); // ❌ 'string' is not assignable to parameter of type 'number' // オプショナル（?）と デフォルト値 const greet = (name: string, greeting?: string): string => { return `${{greeting ?? "こんにちは"}}、${{name}}さん！`; }; // Union型（複数の型を許可） let id: string | number; id = "abc"; // OK id = 123; // OK id = true; // ❌
TypeScript（interfaceとtype）
// interface：オブジェクトの形を定義 interface User { id: number; name: string; email: string; age?: number; // ?で省略可能 readonly createdAt: string; // readonlyで変更不可 } // interfaceを使って型チェック const user: User = { id: 1, name: "田中太郎", email: "taro@example.com", createdAt: "2026-01-01" }; // 関数の引数にinterfaceを使う const showUser = (user: User): string => { return `${{user.name}} (${{user.email}})`; }; // type：より柔軟な型定義 type Priority = "high" | "medium" | "low"; // Union型 type ID = string | number; const setPriority = (p: Priority) => console.log(p); setPriority("high"); // OK setPriority("urgent"); // ❌ 型エラー
TypeScriptはJavaScriptに変換される
TSを学んでもJSの知識は全部活きる
TypeScriptを書く→コンパイルするとJavaScriptが生成されます。ブラウザが実行するのはJavaScriptです。

つまりこれまで学んだJS知識は全部TypeScriptでも使えます。型のアノテーションが追加されただけです。

環境：npx tsc --initでtsconfig作成・VSCodeでリアルタイム型チェックが有効になります。
👆
講師メモTSの全機能を教えるのではなく「型アノテーション」「interface」「Union型」の3つに絞って説明する。深入りするとJSの学習が止まる。「ReactとTypeScriptは同時に学べる」と伝えてモチベーションを保つ。

講師がこう話す「既存のJavaScriptコードにTypeScriptの型を追加してみます。特にAPIのレスポンスに型を付けると、存在しないプロパティへのアクセスをすぐに発見できます。」
TypeScript（API・DOM操作への型付け）
// APIレスポンスに型を付ける interface Post { id: number; title: string; body: string; userId: number; } const fetchPosts = async (): Promise<Post[]> => { const res = await fetch("https://jsonplaceholder.typicode.com/posts"); return res.json(); }; const showPosts = async () => { const posts = await fetchPosts(); posts.forEach(post => { console.log(post.title); // 補完が効く！ // post.author → ❌ 型エラー（Postにauthorはない） }); }; // DOM操作に型を付ける const input = document.querySelector<HTMLInputElement>("#name"); // input?.value が補完される（HTMLInputElement型として認識） const btn = document.querySelector<HTMLButtonElement>("#submit"); btn?.addEventListener("click", () => { console.log(input?.value); });
💡
TypeScriptを始めるには npx tsc --init でtsconfig.jsonを生成し、.ts拡張子でファイルを作るだけです。VSCodeはTypeScriptをネイティブサポートしているので、設定なしでリアルタイムの型チェックと補完が使えます。Playgroundサイト（https://www.typescriptlang.org/play）でブラウザだけでも試せます。
