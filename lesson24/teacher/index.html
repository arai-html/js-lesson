0〜25分
💬
講師がこう話す「AjaxはAsynchronous JavaScript And XMLの略で、ページを全てリロードせずにサーバーとデータをやり取りする技術の総称です。現代ではfetchを使います。RESTful APIとはHTTPメソッド（GET/POST/PUT/DELETE）でリソースを操作するAPIの設計規約です。」
🌐
HTTPメソッドとCRUD操作の対応
GET　→ データを取得（Read）
POST　→ データを新規作成（Create）
PUT/PATCH → データを更新（Update）
DELETE → データを削除（Delete）

JSONPlaceholderで全てのメソッドを練習できます。
script.js（GET・POST・PUT・DELETE）
const BASE = "https://jsonplaceholder.typicode.com"; // GET：データを取得 const getPosts = async () => { const res = await fetch(`${{BASE}}/posts?_limit=5`); const posts = await res.json(); return posts; }; // POST：データを送信して新規作成 const createPost = async (data) => { const res = await fetch(`${{BASE}}/posts`, { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify(data), }); return await res.json(); // 作成されたデータが返る }; // PUT：データを更新 const updatePost = async (id, data) => { const res = await fetch(`${{BASE}}/posts/${{id}}`, { method: "PUT", headers: { "Content-Type": "application/json" }, body: JSON.stringify(data), }); return await res.json(); }; // DELETE：データを削除 const deletePost = async (id) => { const res = await fetch(`${{BASE}}/posts/${{id}}`, { method: "DELETE" }); return res.ok; // true なら成功 };
POSTリクエストの3つのセット
忘れると動かない
POSTやPUTでデータを送るときは必ず3つセットで書きます。

① method: 'POST'
② headers: {{'Content-Type': 'application/json'}}
③ body: JSON.stringify(data)

Content-Typeを忘れるとサーバーがJSONとして読んでくれません。
👆
講師メモJSONPlaceholderはPOST/PUT/DELETEも受け付けるが実際には保存されない（フェイクAPI）。レスポンスを見て「ちゃんと返ってきた！」という体験をさせることが大事。

ローディング・エラー処理・UI反映
25〜45分
💬
講師がこう話す「API通信をUIと組み合わせるときは、ローディング状態・成功状態・エラー状態の3つを管理するのが基本です。この3つの状態を変数で持ちながら画面を更新するパターンを作りましょう。」
script.js（ローディング・エラー処理の完全版）
// UI要素 const listEl = document.querySelector("#post-list"); const loadingEl = document.querySelector("#loading"); const errorEl = document.querySelector("#error"); // 状態を管理する関数 const setState = ({ loading, error, data }) => { loadingEl.hidden = !loading; errorEl.hidden = !error; if (error) errorEl.textContent = error; if (data) listEl.innerHTML = data.map(post => ` <article class="post"> <h3>${{post.title}}</h3> <p>${{post.body}}</p> </article> `).join(""); }; // データを取得してUIに反映 const loadPosts = async () => { setState({ loading: true }); // ローディング開始 try { const res = await fetch(`${{BASE}}/posts?_limit=10`); if (!res.ok) throw new Error(`HTTP ${{res.status}}`); const data = await res.json(); setState({ loading: false, data }); // 成功 } catch (err) { setState({ loading: false, error: `エラー: ${{err.message}}` }); } }; loadPosts();
💡
APIのURLにクエリパラメータでページングやフィルターを指定できます。JSONPlaceholderでは ?_limit=5 で件数制限、?_page=2 でページ指定、?userId=1 でユーザー絞り込みができます。本番のAPIでも同様の仕組みが多いので覚えておきましょう。
