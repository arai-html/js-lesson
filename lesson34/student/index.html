Gitでバージョン管理を始めよう
0〜25分
📖
Gitはコードの変更履歴を管理するツールです。いつでも過去の状態に戻れます。エンジニア必須スキルです。
🌿
Gitの基本概念
リポジトリ：変更履歴を保存する場所
コミット：変更のスナップショット（セーブポイント）
ブランチ：作業の分岐（メインコードを守りながら機能開発）
マージ：ブランチを合流させる
GitHub：Gitリポジトリをクラウドで管理・公開できるサービス
ターミナル（Gitの基本コマンド）
# Gitの初期設定（最初の1回だけ） git config --global user.name "あなたの名前" git config --global user.email "あなたのメール" # ① リポジトリを初期化 git init # ② 現在の状態を確認 git status # ③ ファイルをステージング（コミット対象に追加） git add index.html # 特定のファイル git add . # 全ての変更 # ④ コミット（スナップショットを保存） git commit -m "初回コミット：天気アプリを追加" # ⑤ コミット履歴を確認 git log --oneline # ⑥ 変更の差分を確認 git diff
ターミナル（ブランチとマージ）
# ブランチの操作 git branch # ブランチ一覧 git branch feature/search # ブランチを作成 git checkout feature/search # ブランチを切り替え git checkout -b feature/sort # 作成して切り替え（ショートカット） # mainブランチに戻ってマージ git checkout main git merge feature/search # よくある操作のショートカット git add . && git commit -m "メッセージ" # add＋commit まとめて # .gitignoreでGit管理から除外 # .env・node_modules・.DS_Storeなど echo "node_modules" >> .gitignore echo ".env" >> .gitignore
コミットメッセージの書き方
読んだ人が変更内容を分かる一文
良いメッセージ：
✅ 「検索機能にdebounceを追加」
✅ 「ToDoの削除ボタンのバグを修正」
✅ 「スマホ対応のレスポンシブCSSを追加」

悪いメッセージ：
❌ 「修正」「変更」「update」「fix」

コミット履歴は「開発日誌」です。未来の自分・チームメンバーへのメッセージです。
❓
確認クイズ
ファイルをコミット対象に追加するコマンドは？
git commit

GitHubへpush・GitHub Pagesで公開
25〜45分
📖
GitHubにpushしてGitHub Pagesで公開すれば、誰でもアクセスできるURLでポートフォリオを公開できます。
ターミナル（GitHubへpush）
# ① GitHubでリポジトリを新規作成（GitHub.com → New repository） # ② リモートリポジトリを登録 git remote add origin https://github.com/ユーザー名/リポジトリ名.git # ③ mainブランチをpush git push -u origin main # 以降の変更をpushするとき git add . git commit -m "変更の説明" git push # リモートの変更をpull（チーム開発でよく使う） git pull origin main # よく使うコマンドまとめ # git init → リポジトリを初期化 # git add . → 全変更をステージング # git commit -m "" → コミット # git push → GitHubに送る # git pull → GitHubから取得 # git status → 現在の状態 # git log --oneline → コミット履歴
GitHub Pages の設定手順
── GitHub Pagesの設定 ── 1. GitHubのリポジトリページを開く 2. Settings タブをクリック 3. 左メニューの Pages をクリック 4. Source を "Deploy from a branch" に設定 5. Branch を "main" / "/(root)" に設定 6. Save をクリック 7. 数分後に以下のURLで公開される： https://ユーザー名.github.io/リポジトリ名/ ── 注意点 ── ・index.html が root にあること ・ファイルパスは相対パスを使う ・APIキーをコードに直書きしたままpushしない
💡
GitHub Desktopというアプリを使えばGUIでGitの操作ができます。コマンドに慣れていない間はGitHub Desktopでも全く問題ありません。慣れてきたらターミナルコマンドに移行していきましょう。VSCodeには標準でGitの操作パネルも内蔵されています。
