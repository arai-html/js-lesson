モジュールでコードを分割しよう
0〜25分
💬
講師がこう話す「コードが長くなってきたら、機能ごとにファイルを分割して管理するのがプロの書き方です。JavaScriptのモジュール機能（ES Modules）を使うと、export/importでファイル間で関数や変数を共有できます。」
utils.js（共通関数をまとめたファイル）
// 名前付きエクスポート（複数エクスポートできる） export const add = (a, b) => a + b; export const subtract = (a, b) => a - b; export const PI = 3.14159; export const formatPrice = (price) => { return price.toLocaleString() + "円"; }; // デフォルトエクスポート（ファイルに1つだけ） export default function greet(name) { return `こんにちは、${{name}}さん！`; }
main.js（インポートして使う）
// 名前付きインポート（{}で指定する） import { add, subtract, PI, formatPrice } from "./utils.js"; console.log(add(3, 5)); // 8 console.log(formatPrice(1980)); // "1,980円" console.log(PI); // 3.14159 // デフォルトインポート（名前は自由につけられる） import greet from "./utils.js"; console.log(greet("太郎")); // "こんにちは、太郎さん！" // 全てをまとめてインポート import * as utils from "./utils.js"; console.log(utils.add(1, 2)); // 3
index.html（type=moduleが必要）
<!-- type="module"を付けないとimportが使えない --> <script type="module" src="main.js"></script>
モジュールを使うメリット
コードが整理されて保守しやすくなる
① コードの分割：機能ごとにファイルを分けられる
② 名前の衝突を防ぐ：モジュール内の変数はグローバルに漏れない
③ 再利用しやすい：同じ関数を複数のファイルからimportできる

HTMLファイルに type="module" を忘れずに！
👆
講師メモtype=moduleを忘れてエラーになるパターンが多い。また、ローカルファイルをブラウザで直接開くとCORSエラーになるのでLive Serverを使うよう伝える。

ES2015+の便利な構文まとめ
25〜45分
💬
講師がこう話す「第2回から少しずつ登場してきたES2015以降の便利な構文をここで一気に整理します。現代のJSコードを読むために必須の知識です。」
script.js（ES2015+まとめ）
// ① オプショナルチェーン（?.）← nullでもエラーにならない const user = null; console.log(user?.name); // undefined（エラーにならない） console.log(user?.address?.city); // undefined // ② Null合体演算子（??）← nullまたはundefinedのときだけデフォルト値 const name = null ?? "ゲスト"; // "ゲスト" const age = 0 ?? 25; // 0（0はnullでないので0のまま） const age2 = 0 || 25; // 25（||は0もfalsy扱いするので注意） // ③ ショートハンドプロパティ（変数名とキー名が同じとき省略） const title = "JS入門"; const price = 2800; const book = { title, price }; // { title: title, price: price } と同じ // ④ 配列・オブジェクトの分割代入（応用） const [a, , b] = [1, 2, 3]; // aは1、bは3（2をスキップ） const { x = 10, y = 20 } = { x: 5 }; // x=5、y=20（デフォルト値） // ⑤ Promise.all（複数の非同期処理を並行実行） const [users, posts] = await Promise.all([ fetch("https://jsonplaceholder.typicode.com/users").then(r => r.json()), fetch("https://jsonplaceholder.typicode.com/posts").then(r => r.json()), ]); // 両方完了してから次へ
💡
?.（オプショナルチェーン）はAPIのデータを扱うときに特に重要です。取得したデータのプロパティが存在しない場合でもエラーにならずundefinedを返します。user?.profile?.avatarのように連続して使えます。
