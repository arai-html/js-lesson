WebストレージとURLパラメータ
0〜25分
💬
講師がこう話す「第13回でlocalStorageを学びましたが、今回はより実践的な使い方と状態管理の設計パターンを学びます。URLSearchParamsを使ったURLパラメータの読み書きも、フィルター条件を保存して共有するのに便利なテクニックです。」
script.js（localStorage 実践パターン）
// 汎用的なストレージユーティリティ const storage = { get(key, fallback = null) { try { const item = localStorage.getItem(key); return item ? JSON.parse(item) : fallback; } catch { return fallback; } }, set(key, value) { try { localStorage.setItem(key, JSON.stringify(value)); return true; } catch { console.error("ストレージへの保存に失敗"); return false; } }, remove(key) { localStorage.removeItem(key); }, clear() { localStorage.clear(); } }; // 使い方 storage.set("cart", [{ id: 1, qty: 2 }]); const cart = storage.get("cart", []); console.log(cart); // [{id:1, qty:2}]
script.js（URLSearchParams でフィルター条件をURLに保存）
// URLパラメータを読み書きする // 例：https://example.com/?category=PC&sort=price-asc // URLパラメータを取得 const params = new URLSearchParams(window.location.search); const category = params.get("category") ?? "all"; const sortBy = params.get("sort") ?? "default"; // URLパラメータを更新（ページリロードなし） const updateURL = (newParams) => { const p = new URLSearchParams(newParams); window.history.replaceState({}, "", `?${{p.toString()}}`); }; // フィルター変更時にURLも更新 const onFilterChange = (category, sort) => { updateURL({ category, sort }); renderWithFilter(category, sort); };
データの永続化の選択肢
用途によって使い分ける
localStorage：ブラウザを閉じても保存。設定・ToDoリスト・カート内容など。
sessionStorage：タブを閉じると消える。フォームの一時保存など。
URLパラメータ：URLに状態を持つ。検索・フィルター条件をブックマーク・共有できる。
Cookie：サーバーとのやり取りが必要なとき（認証トークンなど）。

シンプルな状態管理パターン
25〜45分
💬
講師がこう話す「Reactなどのフレームワークが当たり前のように使っている「状態管理」の考え方を、バニラJSで実装してみます。状態（state）を一か所で管理して、変化したら自動でUIを更新するパターンです。フレームワークを学ぶ前の土台として非常に重要です。」
script.js（シンプルなストアパターン）
// シンプルなストア（状態管理）の実装 const createStore = (initialState) => { let state = { ...initialState }; const listeners = new Set(); return { // 状態を取得 getState: () => ({ ...state }), // 状態を更新（変更分だけマージ） setState(updates) { state = { ...state, ...updates }; listeners.forEach(fn => fn(state)); // 全リスナーに通知 }, // 状態変化を購読（戻り値で購読解除） subscribe(fn) { listeners.add(fn); return () => listeners.delete(fn); // 購読解除関数を返す } }; }; // ショッピングカートに使う const store = createStore({ cart: [], wishlist: [], filter: "all" }); // 状態変化を購読してUIを自動更新 store.subscribe((state) => { renderCart(state.cart); renderBadge(state.cart.length); }); // カートに商品を追加 store.setState({ cart: [...store.getState().cart, { id: 1, qty: 1 }] }); // → subscribeのコールバックが自動で呼ばれてUIが更新される
💡
このストアパターンはReduxやZustandなどの状態管理ライブラリと同じ考え方です。「状態を変更する → 自動でUIが更新される」という一方向データフローを理解しておくと、Reactを学ぶときに非常にスムーズに入れます。
