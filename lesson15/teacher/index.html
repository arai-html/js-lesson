classを定義しよう
0〜25分
💬
講師がこう話す「classはオブジェクトの設計図です。同じ構造のオブジェクトをたくさん作るとき、classを使えば毎回プロパティを書かなくて済みます。たとえば社員管理システムで100人の社員オブジェクトを作るとき、classがあれば new Employee('田中', 25) の一行で作れます。」
script.js（classの基本）
// クラスの定義 class Person { // constructor：newしたときに最初に実行される constructor(name, age) { this.name = name; // this = 作られるオブジェクト自身 this.age = age; } // メソッド（全インスタンスで共有される） greet() { return `こんにちは、${{this.name}}（${{this.age}}歳）です`; } birthday() { this.age++; return `${{this.name}}は${{this.age}}歳になりました！`; } } // インスタンスを作る（newを使う） const taro = new Person("田中太郎", 25); const hanako = new Person("山田花子", 22); console.log(taro.greet()); // "こんにちは、田中太郎（25歳）です" console.log(hanako.greet()); // "こんにちは、山田花子（22歳）です" console.log(taro.birthday()); // "田中太郎は26歳になりました！"
script.js（getter・setter）
class Temperature { constructor(celsius) { this._celsius = celsius; // _は「直接触らないで」という慣習 } // getter：プロパティのように値を取得できる get fahrenheit() { return this._celsius * 9 / 5 + 32; } // setter：プロパティのように値をセットできる set celsius(value) { if (value < -273.15) throw new Error("絶対零度以下は不正"); this._celsius = value; } } const temp = new Temperature(100); console.log(temp.fahrenheit); // 212（メソッドっぽく見えないが計算される） temp.celsius = 37; // setterが呼ばれる
thisの意味
インスタンス自身を指す
クラスのメソッド内の this は、そのメソッドを呼び出したインスタンスを指します。

taro.greet() のとき this は taro
hanako.greet() のとき this は hanako

thisは混乱しやすいポイントですが「自分自身」と覚えれば大丈夫です。
👆
講師メモthisの混乱はJSあるある。アロー関数のthisはクラスでは使わないことを伝える（bindが必要になる）。今は「メソッドの中のthisは自分自身」でOK。

継承・static・プライベートフィールド
25〜45分
💬
講師がこう話す「継承はあるクラスの機能を引き継いで新しいクラスを作る仕組みです。extendsを使います。共通の機能を親クラスに書いて、子クラスで独自機能を追加できます。」
script.js（継承・static・プライベート）
// 親クラス class Animal { constructor(name) { this.name = name; } speak() { return `${{this.name}}が鳴いた`; } } // 子クラス（Animalを継承） class Dog extends Animal { constructor(name, breed) { super(name); // 親のconstructorを呼ぶ（必須） this.breed = breed; } // メソッドのオーバーライド（上書き） speak() { return `${{this.name}}（${{this.breed}}）がワンワン！`; } fetch() { return `${{this.name}}がボールを取ってきた！`; } } const dog = new Dog("ポチ", "柴犬"); console.log(dog.speak()); // "ポチ（柴犬）がワンワン！" console.log(dog.fetch()); // "ポチがボールを取ってきた！" // staticメソッド：インスタンスを作らず直接呼べる class MathUtils { static square(x) { return x * x; } static cube(x) { return x * x * x; } } console.log(MathUtils.square(4)); // 16 // プライベートフィールド（#）：クラス外からアクセス不可 class BankAccount { #balance = 0; // #で始まるとプライベート deposit(amount) { this.#balance += amount; } get balance() { return this.#balance; } } const acc = new BankAccount(); acc.deposit(1000); console.log(acc.balance); // 1000 // console.log(acc.#balance); // ❌ SyntaxError！
