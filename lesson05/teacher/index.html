関数を定義しよう
0〜25分
💬
講師がこう話す「関数は処理をまとめて名前をつけたものです。同じ処理を何度も書かなくていいし、後から修正するときも1か所直すだけで済みます。関数はJavaScriptの中核です。書き方が3種類ありますが、現代では主にアロー関数が使われます。」
script.js（関数の3つの書き方）
// ① 関数宣言（function文） function greet(name) { return `こんにちは、${{name}}さん！`; } console.log(greet("太郎")); // "こんにちは、太郎さん！" // ② 関数式（変数に関数を代入） const greet2 = function(name) { return `こんにちは、${{name}}さん！`; }; console.log(greet2("花子")); // ③ アロー関数（現代の書き方・最もよく使う） const greet3 = (name) => { return `こんにちは、${{name}}さん！`; }; // 処理が1行のときはさらに省略できる const greet4 = name => `こんにちは、${{name}}さん！`; console.log(greet4("次郎")); // "こんにちは、次郎さん！"
script.js（引数・デフォルト値・戻り値）
// 引数が複数のとき const add = (a, b) => a + b; console.log(add(3, 5)); // 8 // デフォルト引数（引数が渡されなかったときの初期値） const greet = (name = "ゲスト") => `こんにちは、${{name}}さん！`; console.log(greet()); // "こんにちは、ゲストさん！" console.log(greet("太郎")); // "こんにちは、太郎さん！" // 戻り値のない関数（副作用だけ行う） const showResult = (score) => { const grade = score >= 80 ? "合格" : "不合格"; console.log(`${{score}}点 → ${{grade}}`); // returnがない → undefinedを返す }; showResult(75); // "75点 → 不合格"
アロー関数の省略ルール
引数と処理の数によって省略できる
① 引数が1つ → カッコ省略可能：name => ...
② 処理が1行のreturnだけ → {}とreturnを省略可能：name => \`こんにちは${name}\`
③ 引数が0個 → カッコ必須：() => ...
④ 引数が2個以上 → カッコ必須：(a, b) => ...
👆
講師メモアロー関数の省略形は最初は混乱しやすい。まず{}とreturnを書いた完全な形を覚えてから省略形を紹介するとスムーズ。

スコープを理解しよう
25〜40分
💬
講師がこう話す「スコープとは変数が使える範囲のことです。関数の中で作った変数は外から使えません。この仕組みを理解しておかないと「変数が見つからない」というエラーでよくハマります。」
script.js（スコープ）
// グローバルスコープ：ファイルのどこからでもアクセス可能 const globalName = "グローバル"; function testScope() { // ローカルスコープ：この関数の中だけで有効 const localName = "ローカル"; console.log(globalName); // ✅ グローバル変数は中から見える console.log(localName); // ✅ ローカル変数は中から見える } testScope(); console.log(globalName); // ✅ グローバル変数は外から見える // console.log(localName); // ❌ ReferenceError！外からは見えない
💡
グローバル変数（関数の外で定義した変数）は便利ですが、使いすぎると「どこで変更されたか分からない」バグの原因になります。変数はなるべく使う場所の近くで定義するのがベストプラクティスです。
